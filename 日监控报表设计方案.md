# 日监控报表功能设计方案

## 一、功能概述

在扣子应用中添加日监控报表功能，包含：
1. **表一：核心经营指标** - 销售额、毛利、毛利率等核心数据
2. **表二：销售目标进度表** - 21个业务经理的业绩进度
3. **亏损订单查询** - 筛选和查询亏损订单

## 三、页面设计

### 3.1 路由结构
```
/src/app/
  /dashboard/
    page.tsx           # 主页面（产品/客户检索）
    daily-report/
      page.tsx         # 日监控报表页面
    loss-orders/
      page.tsx         # 亏损订单查询页面
```

### 3.2 组件结构
```
/src/components/
  DailyReport/
    IndicatorsCard.tsx    # 核心指标卡片
    SalesProgressTable.tsx # 销售进度表格
    MarginZonesChart.tsx  # 毛利区间图表
  LossOrders/
    FilterPanel.tsx       # 筛选面板
    OrdersTable.tsx       # 订单表格
```

## 四、API设计

### 4.1 获取日监控数据
```
GET /api/daily-report?date=2026-04-17
```

### 4.2 获取亏损订单
```
GET /api/loss-orders?startDate=2026-04-01&endDate=2026-04-17&salesman=刘文
```

## 五、数据更新机制

### 5.1 定时任务
每天8点自动运行日监控模型，生成最新数据

### 5.2 数据存储
- 日监控数据存储在 `/src/data/daily-report.json`
- 亏损订单数据存储在 `/src/data/loss-orders.json`

## 六、实现步骤

1. **创建数据文件** - 在扣子项目中创建数据存储文件
2. **添加页面路由** - 创建新的页面组件
3. **实现数据展示** - 展示核心指标和销售进度表
4. **实现亏损订单查询** - 筛选和查询功能
5. **对接定时任务** - 每天自动更新数据
