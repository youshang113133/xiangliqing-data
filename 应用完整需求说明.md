# 扣子应用自动化运营需求说明

## 一、应用概述

这是一个基于扣子平台的业务管理系统，包含产品检索、客户检索、日监控报表三大核心功能，需要实现每日自动化更新和部署。

---

## 二、核心功能需求

### 1. 日监控报表（每日自动生成）

**功能目标**：每天自动生成核心经营指标报表

**报表内容**：
- **表一：核心经营指标**
  - 销售额（当日/月累计）
  - 毛利额（当日/月累计）
  - 毛利率（当日/月累计）
  - 订单数
  - 亏损订单数
  - 亏损金额

- **表二：销售目标进度表**
  - 21个业务经理的业绩进度
  - 月度目标完成率
  - 同期对比
  - 毛利率达标情况

- **表三：亏损订单查询**
  - 按日期、业务员、客户筛选
  - 显示亏损订单明细

**数据来源**：
- T+系统销货单明细
- 每天早上8点自动抓取前一天数据

**输出文件**：
- `./日销售监控/YYYY-MM-DD_日销售监控报告.xlsx`

---

### 2. 产品检索（每日自动更新）

**功能目标**：提供产品信息检索，重点展示库存变化

**核心字段**：

#### 基础信息（静态）
- 产品编号
- 产品名称
- 规格型号
- 产品类型
- 主要对接业务员

#### 动态信息（每日更新）
- **库存数量变化**（重点）
  - 当前库存数量（件）
  - 库存变化量（件）
  - 计算公式：**数量/规格 = 件数**
  - 变化趋势：📈 上升 / 📉 下降 / ➡️ 稳定

- 库存金额
- 动销状态
- 库存周转天数

**库存变化计算逻辑**：
```
1. 获取前一天的销货单明细
2. 统计每个产品的销售数量
3. 获取总库存表中的当前库存
4. 计算：
   - 今日库存件数 = 总库存数量 ÷ 规格数量
   - 昨日库存件数 = 今日库存 + 昨日销售数量 ÷ 规格数量
   - 库存变化 = 今日库存件数 - 昨日库存件数
   - 变化率 = (今日库存 - 昨日库存) / 昨日库存
```

**数据来源**：
- T+系统销货单明细（每日）
- T+系统总库存表（每日）

---

### 3. 客户检索（每日自动更新）

**功能目标**：提供客户信息检索，重点展示预计进货时间和跟进建议

**核心字段**：

#### 基础信息（静态）
- 客户编号
- 客户名称
- 所属区域
- 联系人
- 联系电话
- 主要对接业务员
- ABCD客户分层

#### 动态信息（每日更新）
- **预计下次进货时间**（重点）
  - 计算公式：**最新采购日期 + (30天 ÷ 月均采购频次)**
  - 基于最新销货单数据动态更新
  
- **跟进建议**（重点）
  - 已超期（≤7天）：显示"已超期，马上跟进（超期X天）"
  - 严重超期（8-30天）：显示"严重超期，紧急跟进"
  - 长期超期（>30天）：显示"长期超期，重点关注"
  - 即将采购（≤3天）：显示"即将采购，提前联系"
  - 近期采购（≤7天）：显示"近期采购，准备跟进"
  - 正常：显示"正常"

- 最近一次采购日期
- 月均采购额
- 月均采购频次
- 近30天采购额
- 采购额变化率

**预计进货时间计算逻辑**：
```
1. 从最近60天的销货单中获取客户最新采购日期
2. 获取客户的月均采购频次
3. 计算：
   - 预计下次进货时间 = 最新采购日期 + timedelta(days=30/月均采购频次)
   - 超期天数 = 今天 - 预计下次进货时间
```

**数据来源**：
- T+系统销货单明细（最近60天）
- 客户画像基础数据

## 三、自动化流程设计

### 每日自动化任务流程

```
时间：每天早上8点

【第一阶段：数据抓取】
1. 从T+系统抓取昨天的销货单明细
   - 登录T+系统（处理登录冲突）
   - 导出销货单明细表
   - 保存到 ./T+数据/YYYY-MM-DD_销货单明细.xlsx

2. 从T+系统抓取当前总库存表
   - 导出库存余额表
   - 保存到 ./T+数据/YYYY-MM-DD_总库存表.xlsx

【第二阶段：数据处理】
3. 计算产品库存变化
   - 对比昨日销货单和总库存表
   - 计算每个产品的库存件数变化
   - 更新 products.json

4. 计算客户动态字段
   - 加载最近60天销货单数据
   - 计算预计下次进货时间
   - 生成跟进建议
   - 更新 customers.json

5. 生成日监控报告
   - 运行日监控模型
   - 生成 ./日销售监控/YYYY-MM-DD_日销售监控报告.xlsx

【第三阶段：自动部署】
6. 部署扣子项目
   - 提交代码变更
   - 触发构建
   - 自动部署

7. 发送通知
   - 发送日监控报告给用户
   - 通知数据更新完成
```

```python
# 伪代码
def calculate_inventory_change(product_id, yesterday_sales, current_inventory):
    spec_qty = get_spec_quantity(product_id)  # 规格数量
    current_pieces = current_inventory / spec_qty
    yesterday_pieces = current_pieces + (yesterday_sales / spec_qty)
    change = current_pieces - yesterday_pieces
    change_rate = change / yesterday_pieces
    return current_pieces, change, change_rate
```
