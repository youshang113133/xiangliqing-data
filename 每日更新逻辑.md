## 三、更新步骤

### 第1步：登录T+导出销货单
- 导航：智能供应链 → 销售管理 → 销货单明细表
- 查询条件：单据日期 = 昨日
- 点击"导出"按钮下载Excel
- **注意**：末尾有"合计"行，必须过滤掉！

### 第2步：计算核心指标
```
当日销售额 = SUM(销货单明细.金额) 排除合计行
当日成本 = SUM(销货单明细.成本) 排除合计行
当日毛利额 = 当日销售额 - 当日成本
当日毛利率 = 当日毛利额 / 当日销售额
当日订单数 = COUNT(DISTINCT 单据编号)

月累计销售额 = 前日累计销售额 + 当日销售额
月累计成本 = 前日累计成本 + 当日成本
月累计毛利额 = 前日累计毛利额 + 当日毛利额
```

### 第3步：更新 data.json 各模块

#### ① dailyReportData（日报数据）
- date：更新为昨日日期
- 核心经营指标.销售额.当日、月累计
- 核心经营指标.成本.当日、月累计
- 核心经营指标.毛利额.当日、月累计
- 毛利区间：按毛利率分布计算
- 销售进度：各业务经理月销售额

#### ② salesProgressData（销售进度）
- 每个业务经理的月销售额（按销货单业务员汇总）
- 完成进度 = 月销售额 / 月度目标
- 当日毛利率、月累计毛利率
- **公司合计 = 21个业务经理销售额之和**

#### ③ productsData（产品数据）
- sales：累计销售额（新增当日销售额）
- salesTimes：销售次数+1
- sellerCount：客户数（仅新客户+1）
- stock：库存 = 存货总账基准 - 累计销售数量÷规格

#### ④ customersData（客户数据）
- lastDate：更新为销货单日期（当日采购的客户）
- nextDate：根据客户等级预测
  - A类客户：lastDate + 15天
  - B类客户：lastDate + 20天
  - 其他客户：lastDate + 30天
- totalPurchase：累计采购额 + 当日采购额

#### ⑤ allCustomers
- 从customersData筛选生成，用于客户决策模块

## 四、核算检查（15项）

| # | 检查项 | 要求 |
|---|--------|------|
| 1 | 成本 + 毛利额 = 销售额 | 必须相等 |
| 2 | 毛利率计算 | abs(毛利率 - 毛利额/销售额) < 0.01 |
| 3 | 各区域小计 | = 该区域业务员之和 |
| 4 | 公司合计 | = 各区域小计之和 |
| 5 | 公司合计验证 | salesProgressData公司合计 ≈ dailyReportData月累计（差值<100）|
| 6 | 过滤总计行 | 销货单末尾合计行必须过滤 |
| 7 | 月累计一致 | 前日累计 + 当日增量 |
| 8 | sellerCount | 只对新客户+1 |
| 9 | lastDate | 当日采购的客户都要更新 |
| 10 | allCustomers | 不能为空 |
| 11 | salesProgressData | 每个业务经理都要更新 |
| 12 | updateTime | 更新为当前时间 |
| 13 | 库存 | 不能为负数 |
| 14 | 毛利率范围 | -20%~50% |
