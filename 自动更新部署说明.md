# 湘里情智智营·客户决策系统

## 自动更新部署说明

### 一、系统架构

```
T+系统 → 定时抓取脚本 → GitHub仓库 → GitHub Pages → 用户访问
```

### 二、目录结构

```
xiangliqing-data/
├── index.html          # 主页面
├── data.json           # 核心数据文件
├── data_core.json      # 核心经营指标
├── data_recs.json      # 推荐数据
├── daily_update.json   # 每日更新状态
├── daily_sales_*.json  # 历史销售数据
├── salesman_margin_range.json  # 业务员毛利率区间
├── tplus_fetch.py      # T+数据抓取脚本
└── 任务管理_使用手册.md  # 本文档
```

### 三、T+数据抓取脚本

#### 3.1 功能说明

`tplus_fetch.py` 脚本实现以下功能：

1. **自动获取T+访问令牌** - 通过API认证获取访问权限
2. **抓取销售明细** - 按日期获取销货单数据
3. **数据解析整理** - 统一数据格式，便于前端使用
4. **更新数据文件** - 生成每日销售JSON文件
5. **提交到GitHub** - 自动git commit和push

#### 3.2 配置步骤

**Step 1: 安装依赖**

```bash
pip3 install requests
```

**Step 2: 配置T+系统连接**

编辑 `tplus_fetch.py`，修改 `TPLUS_CONFIG` 配置：

```python
TPLUS_CONFIG = {
    "api_url": "http://你的T+服务器地址/api",
    "app_key": "你的APP_KEY",
    "app_secret": "你的APP_SECRET",
    "account": "你的账套代码"
}
```

**Step 3: 配置Git提交信息**

```python
GIT_EMAIL = "bot@xiangliqing.com"
GIT_NAME = "XLQ Bot"
```

**Step 4: 测试运行**

```bash
# 测试今日数据抓取
python3 tplus_fetch.py

# 测试指定日期
python3 tplus_fetch.py --date 2026-05-01
```

### 四、定时任务配置

#### 4.1 Linux/Mac (使用 crontab)

```bash
# 编辑crontab
crontab -e

# 添加定时任务（每日8点执行）
0 8 * * * cd /path/to/xiangliqing-data && /usr/bin/python3 tplus_fetch.py >> /var/log/tplus_fetch.log 2>&1
```

#### 4.2 Windows (使用任务计划程序)

1. 打开"任务计划程序"
2. 创建基本任务
3. 设置触发器：每日 8:00
4. 设置操作：启动程序，程序路径选择 `python.exe`，参数填写 `tplus_fetch.py`

#### 4.3 GitHub Actions (推荐方案)

在仓库中创建 `.github/workflows/daily_update.yml`：

```yaml
name: Daily T+ Data Update

on:
  schedule:
    - cron: '0 0 * * *'  # 每天UTC 0点 (北京时间8点)
  workflow_dispatch:       # 支持手动触发

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          token: ${{ secrets.GH_TOKEN }}
      
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: Install dependencies
        run: pip install requests
      
      - name: Fetch T+ data
        env:
          TPLUS_API_URL: ${{ secrets.TPLUS_API_URL }}
          TPLUS_APP_KEY: ${{ secrets.TPLUS_APP_KEY }}
          TPLUS_APP_SECRET: ${{ secrets.TPLUS_APP_SECRET }}
          TPLUS_ACCOUNT: ${{ secrets.TPLUS_ACCOUNT }}
        run: python3 tplus_fetch.py
      
      - name: Commit and push
        run: |
          git config user.email "bot@xiangliqing.com"
          git config user.name "XLQ Bot"
          git add daily_sales_*.json daily_update.json data.json
          git commit -m "📊 数据更新 $(date +%Y-%m-%d)" || echo "No changes"
          git push
```

需要在 GitHub 仓库设置中添加以下 Secrets：
- `GH_TOKEN`: GitHub Personal Access Token
- `TPLUS_API_URL`: T+ API地址
- `TPLUS_APP_KEY`: T+应用Key
- `TPLUS_APP_SECRET`: T+应用密钥
- `TPLUS_ACCOUNT`: 账套代码

### 七、故障排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 无法获取Token | T+系统配置错误 | 检查API地址和认证信息 |
| 无销售数据 | 非工作日 | 确认日期是否为工作日 |
| Git推送失败 | Token过期 | 更新GitHub Personal Access Token |
| 定时任务不执行 | crontab配置错误 | 检查crontab语法 |

### 八、注意事项

1. **数据安全**: T+系统的API凭证请妥善保管，不要提交到公开仓库
2. **频率限制**: 建议每日执行一次，避免对T+系统造成压力
3. **数据备份**: 重要数据建议定期备份
4. **错误监控**: 建议配置邮件或钉钉通知，及时发现异常
